Chapter P — Security Hardening

AreaRoutine
Apache Security 

Edit httpd.conf:
- Disable directory listing: Options -Indexes
- Restrict access: Require all denied (except htdocs)
- Enable mod_ssl for HTTPS
- Use strong TLS protocols: TLSv1.2, TLSv1.3
- Configure firewall to allow only port 80/443
PHP Security 

Edit php.ini:
- expose_php = Off
- display_errors = Off
- log_errors = On
- disable_functions = exec, system, shell_exec, passthru
- session.cookie_httponly = 1
- session.cookie_secure = 1
GitHub Security 

Enable 2FA (Authenticator app)
Use SSH keys for repo access:
ssh-keygen -t ed25519 -C "academy@example.com"
Add public key to GitHub → Settings → SSH Keys
Verify: ssh -T git@github.com
WinSCP Security 

Preferences → Security:
- Disable plain FTP, use SFTP/SSH
- Enable password manager encryption
- Use key-based authentication
- Enable session logging for audits
System Security 

Windows Defender:
- Enable real-time protection
- Schedule weekly scans
Firewall:
- Block unused ports
- Allow only Apache, WinSCP, GitHub CLI
