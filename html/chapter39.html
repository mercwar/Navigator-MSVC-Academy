Chapter 39 — Engine Architecture: Subsystems, Messaging, Memory Pools & Game Loop

Topic Description
Engine Architecture Overview A game engine is composed of independent subsystems:
  • Rendering
  • Physics
  • AI
  • Audio
  • Input
  • Networking
  • Resource management
  • Scene graph
These systems communicate through messaging or shared state.
Subsystem Structure 

struct Subsystem {
    virtual void Init() = 0;
    virtual void Update(float dt) = 0;
    virtual void Shutdown() = 0;
};
Example: 

class PhysicsSystem : public Subsystem {
public:
    void Init() override;
    void Update(float dt) override;
    void Shutdown() override;
};
Engine Core 

class Engine {
public:
    Renderer renderer;
    PhysicsSystem physics;
    AudioSystem audio;
    AISystem ai;
    NetworkSystem net;

    void Init() {
        renderer.Init();
        physics.Init();
        audio.Init();
        ai.Init();
        net.Init();
    }

    void Update(float dt) {
        net.Update(dt);
        input.Update(dt);
        ai.Update(dt);
        physics.Update(dt);
        renderer.Update(dt);
        audio.Update(dt);
    }

    void Shutdown() {
        audio.Shutdown();
        renderer.Shutdown();
        physics.Shutdown();
        ai.Shutdown();
        net.Shutdown();
    }
};
Messaging System Subsystems communicate via message bus: 

struct Message {
    int type;
    void *data;
};

class MessageBus {
public:
    vector queue;

    void Send(Message m) {
        queue.push_back(m);
    }

    void Dispatch() {
        for (auto &m : queue)
            Deliver(m);
        queue.clear();
    }
};
Example message: 

Message m;
m.type = MSG_COLLISION;
m.data = &collisionInfo;
bus.Send(m);
Memory Pools Memory pools reduce fragmentation and speed allocation. 

struct Pool {
    char *mem;
    size_t size;
    size_t offset;
};

void *PoolAlloc(Pool &p, size_t s) {
    void *ptr = p.mem + p.offset;
    p.offset += s;
    return ptr;
}
Reset each frame: 

pool.offset = 0;
Perfect for:
  • Particles
  • Temporary buffers
  • Frame‑based allocations
Job System (Multithreading) Job systems distribute work across CPU cores. Job: 

struct Job {
    void (*func)(void*);
    void *data;
};
Worker thread: 

void Worker() {
    while (running) {
        Job j = queue.Pop();
        j.func(j.data);
    }
}
Submit job: 

Job j;
j.func = UpdatePhysicsChunk;
j.data = chunkPtr;
queue.Push(j);
Used for:
  • Physics
  • AI
  • Animation
  • Occlusion culling
Scene Graph Scene graph organizes objects hierarchically. 

struct Node {
    XMFLOAT3 pos;
    XMFLOAT3 rot;
    XMFLOAT3 scale;
    vector children;
};
Update: 

void UpdateNode(Node *n, XMMATRIX parent) {
    XMMATRIX local = CalcMatrix(n);
    XMMATRIX world = XMMatrixMultiply(local, parent);

    for (auto *c : n->children)
        UpdateNode(c, world);
}
Resource Manager Resources:
  • Textures
  • Meshes
  • Shaders
  • Audio

unordered_map cache;

Resource *Load(string path) {
    if (cache.count(path))
        return cache[path];

    Resource *r = LoadFromDisk(path);
    cache[path] = r;
    return r;
}
Game Loop 

while (running) {
    float dt = timer.GetDelta();

    input.Update(dt);
    engine.Update(dt);
    engine.renderer.Render();
}
Deterministic physics: 

accumulator += dt;

while (accumulator >= fixedStep) {
    physics.Update(fixedStep);
    accumulator -= fixedStep;
}
Event System 

struct Event {
    int type;
    void *payload;
};

vector events;

void Emit(Event e) {
    events.push_back(e);
}

void ProcessEvents() {
    for (auto &e : events)
        HandleEvent(e);
    events.clear();
}
