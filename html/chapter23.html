Chapter 23 — DLLs, Dynamic Loading & Plugin Architecture

Topic Description
What is a DLL? A DLL (Dynamic Link Library) is a module containing functions and data that can be used by multiple programs. DLLs allow:
  • Modular architecture
  • Shared code
  • Plugins
  • Runtime loading/unloading
DLLs use the .dll extension.
Exporting Functions DLLs export functions using __declspec(dllexport): 

// mydll.c
__declspec(dllexport)
int Add(int a, int b)
{
    return a + b;
}
Header for consumers: 

// mydll.h
__declspec(dllimport)
int Add(int a, int b);
Exporting with .def File Instead of dllexport, you can use a module definition file: 

LIBRARY "mydll"
EXPORTS
    Add
    Sub
This is useful for controlling export names.
DllMain DLLs have an optional entry point: 

BOOL WINAPI DllMain(HINSTANCE hInst,
                    DWORD reason,
                    LPVOID reserved)
{
    switch (reason) {
        case DLL_PROCESS_ATTACH:
            // loaded
            break;
        case DLL_PROCESS_DETACH:
            // unloaded
            break;
    }
    return TRUE;
}
DllMain runs automatically when the DLL is loaded/unloaded.
Static Linking to DLLs Link with the import library: 

#pragma comment(lib, "mydll.lib")
#include "mydll.h"

int r = Add(2, 3);
The DLL must be present at runtime.
Dynamic Loading (LoadLibrary) Load DLL at runtime: 

HMODULE h = LoadLibrary("mydll.dll");
Get function pointer: 

typedef int (*AddFunc)(int, int);

AddFunc Add = (AddFunc)GetProcAddress(h, "Add");

int r = Add(5, 7);
Unload: 

FreeLibrary(h);
Dynamic loading enables plugin systems.
Plugin Architecture Plugins are DLLs loaded at runtime. Common pattern: 

// Plugin interface
typedef void (*PluginInit)(void);
typedef void (*PluginRun)(void);
DLL exports: 

__declspec(dllexport) void PluginInit(void) { ... }
__declspec(dllexport) void PluginRun(void)  { ... }
Host application: 

HMODULE h = LoadLibrary("plugin.dll");

PluginInit init = (PluginInit)GetProcAddress(h, "PluginInit");
PluginRun  run  = (PluginRun) GetProcAddress(h, "PluginRun");

init();
run();

FreeLibrary(h);
This allows unlimited extensibility.
Resource‑Only DLLs DLLs can contain only resources (icons, dialogs, strings): 

LoadString(hResDll, IDS_HELLO, buffer, 256);
Useful for localization.
Side‑by‑Side (SxS) DLL Loading Windows can load different versions of the same DLL using manifests. 

<assembly>
  <dependency>
    <dependentAssembly>
      <assemblyIdentity name="MyDLL" version="1.0.0.0"/>
    </dependentAssembly>
  </dependency>
</assembly>
Prevents “DLL Hell”.
Full Dynamic Loading Example 

// main.c
#include <windows.h>
#include <stdio.h>

typedef int (*AddFunc)(int, int);

int main(void)
{
    HMODULE h = LoadLibrary("mydll.dll");
    if (!h) {
        printf("Failed to load DLL\n");
        return 1;
    }

    AddFunc Add = (AddFunc)GetProcAddress(h, "Add");
    if (!Add) {
        printf("Function not found\n");
        return 1;
    }

    printf("Result: %d\n", Add(10, 20));

    FreeLibrary(h);
    return 0;
}
