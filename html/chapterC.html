Chapter C — CYHY: Diagnostics, Telemetry & DirectX Health

TopicDescription
CYHY Role
  • CYHY = health monitor + telemetry for all subsystems.
  • Watches DX device, swap chain, factories, and render targets.
  • Feeds AVIS with status flags and error codes.
DX Health Snapshot 

struct DXHealth {
    bool deviceLost;
    bool swapValid;
    bool d2dReady;
    bool dwriteReady;
    HRESULT lastError;
};

DXHealth CYHY_CheckDX(const DXVector &dx)
{
    DXHealth h = {};
    h.swapValid  = (dx.swap != nullptr);
    h.d2dReady   = (dx.d2dFactory != nullptr);
    h.dwriteReady= (dx.dwFactory != nullptr);
    // deviceLost via DXGI or test draw...
    return h;
}
AVIS Integration 

void CYHY_Update()
{
    DXHealth h = CYHY_CheckDX(avis.cbord->dx);
    avis.dxHealth = h;

    if (h.deviceLost)
        avis.flags |= AVIS_FLAG_DX_REBUILD;
}
Recovery Vector
  • CYHY sets flags; CBORD performs DX rebuild.
  • CYBORG pauses command execution until AVIS clears error.
  • All DirectX interaction is gated by CYHY health.
