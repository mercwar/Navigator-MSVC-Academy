Chapter A — AVIS Core & CBORD: Subsystem Vector Map to DirectX

TopicDescription
AVIS Core
  • AVIS = global diagnostic + boot matrix for MercWar engine.
  • Tracks subsystems: CBORD, CYBORG, CYHY, MERC‑G.
  • Each subsystem exposes a vector table of capabilities.
CBORD Role
  • CBORD = low‑level OS/WinAPI + DirectX binding layer.
  • Owns window class, message pump, and DX device bootstrap.
  • Exports handles: HWND, HINSTANCE, DXGI factory, D3D device.
Subsystem Vector Table 

struct DXVector {
    ID3D11Device          *d3d;
    ID3D11DeviceContext   *ctx;
    IDXGISwapChain        *swap;
    ID2D1Factory          *d2dFactory;
    IDWriteFactory        *dwFactory;
};

struct CBORD {
    HWND      hwnd;
    HINSTANCE hinst;
    DXVector  dx;
};
AVIS Registration 

void AVIS_Register_CBORD(CBORD *cbord)
{
    avis.cbord = cbord;
    avis.flags |= AVIS_FLAG_CBORD_READY;
}
DirectX Bootstrap CBORD owns DX init, then exposes it upward: 

void CBORD_InitDX(CBORD *c)
{
    // create device, swap chain, factories...
    c->dx.d3d  = device;
    c->dx.ctx  = context;
    c->dx.swap = swap;
