Chapter 38 — Networking: Sockets, Client/Server, Replication & Multiplayer Architecture

Topic Description
Networking Overview Real‑time networking requires:
  • Socket communication
  • Client/server architecture
  • Replication of game state
  • Lag compensation
  • Prediction & interpolation
Windows uses **WinSock** for networking.
WinSock Initialization 

WSADATA wsa;
WSAStartup(MAKEWORD(2,2), &wsa);
Cleanup: 

WSACleanup();
TCP Socket Create: 

SOCKET s = socket(AF_INET, SOCK_STREAM, IPPROTO_TCP);
Connect: 

sockaddr_in addr = {0};
addr.sin_family = AF_INET;
addr.sin_port = htons(9000);
addr.sin_addr.s_addr = inet_addr("127.0.0.1");

connect(s, (sockaddr*)&addr, sizeof(addr));
Send: 

send(s, "Hello", 5, 0);
Receive: 

recv(s, buffer, 512, 0);
TCP Server 

SOCKET server = socket(AF_INET, SOCK_STREAM, IPPROTO_TCP);

sockaddr_in addr = {0};
addr.sin_family = AF_INET;
addr.sin_port = htons(9000);
addr.sin_addr.s_addr = INADDR_ANY;

bind(server, (sockaddr*)&addr, sizeof(addr));
listen(server, SOMAXCONN);

SOCKET client = accept(server, NULL, NULL);
UDP Socket 

SOCKET u = socket(AF_INET, SOCK_DGRAM, IPPROTO_UDP);
Send: 

sendto(u, data, len, 0, (sockaddr*)&addr, sizeof(addr));
Receive: 

recvfrom(u, buffer, 512, 0, NULL, NULL);
UDP is preferred for real‑time games.
Non‑Blocking Mode 

u_long mode = 1;
ioctlsocket(s, FIONBIO, &mode);
Prevents blocking the game loop.
Socket Multiplexing (select) 

fd_set set;
FD_ZERO(&set);
FD_SET(s, &set);

timeval tv = {0, 0};

if (select(0, &set, NULL, NULL, &tv) > 0) {
    recv(s, buffer, 512, 0);
}
Allows checking multiple sockets.
Client/Server Architecture Server responsibilities:
  • Authoritative game state
  • Collision & physics
  • Replication
  • Anti‑cheat
Client responsibilities:
  • Input
  • Prediction
  • Interpolation
  • Rendering
Replication Replicate game objects: 

struct RepPacket {
    int id;
    float x, y, z;
    float vx, vy, vz;
};
Server sends: 

sendto(u, &packet, sizeof(packet), 0, ...);
Client applies: 

objects[id].pos = Lerp(objects[id].pos, packet.pos, 0.1f);
Client‑Side Prediction Predict movement before server confirms: 

predictedPos += inputDir * speed * dt;
When server update arrives: 

error = serverPos - predictedPos;
predictedPos += error * 0.1f;
Smooth correction prevents jitter.
Interpolation Interpolate between past states: 

renderPos = Lerp(prevPos, nextPos, alpha);
Alpha = time offset between packets.
Lag Compensation Server rewinds world to past time: 

float rewindTime = clientPing * 0.5f;
SimulateTo(time - rewindTime);
Used for hit detection in shooters.
Packet Loss Handling Use sequence numbers: 

struct Packet {
    int seq;
    float x,y,z;
};
Client ignores old packets: 

if (packet.seq < lastSeq) return;
Full Multiplayer Loop 

// CLIENT
SendInput();
Predict();
ReceiveUpdates();
Interpolate();
Render();

// SERVER
ReceiveInputs();
SimulatePhysics();
BroadcastState();
