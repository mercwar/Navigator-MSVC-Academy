Chapter 12 — Introduction to the Windows API (WinAPI)

Topic Description
What is WinAPI? The Windows API (WinAPI) is the core interface between applications and the Windows operating system. It provides thousands of functions for:
  • Creating windows
  • Handling messages
  • Drawing graphics (GDI)
  • Managing files and directories
  • Working with processes and threads
  • Interacting with hardware
WinAPI is written in C and designed for low-level control.
Including WinAPI Every WinAPI program begins with: 

#include <windows.h>
This header pulls in:
  • winuser.h — window creation & messages
  • wingdi.h — drawing
  • winbase.h — core system functions
  • commctrl.h — common controls
WinMain — the WinAPI entry point GUI applications use WinMain instead of main(): 

int WINAPI WinMain(HINSTANCE hInstance,
                   HINSTANCE hPrevInstance,
                   LPSTR     lpCmdLine,
                   int       nCmdShow)
{
    return 0;
}
Parameters:
  • hInstance — handle to the current program
  • lpCmdLine — command-line string
  • nCmdShow — window display state
Window classes Before creating a window, you must register a window class: 

WNDCLASSEX wc = {0};
wc.cbSize        = sizeof(WNDCLASSEX);
wc.lpfnWndProc   = WndProc;
wc.hInstance     = hInstance;
wc.lpszClassName = "MyWindowClass";

RegisterClassEx(&wc);
WndProc is the callback that handles window messages.
CreateWindowEx After registering the class, create the window: 

HWND hwnd = CreateWindowEx(
    0,
    "MyWindowClass",
    "My First WinAPI Window",
    WS_OVERLAPPEDWINDOW,
    CW_USEDEFAULT, CW_USEDEFAULT,
    800, 600,
    NULL, NULL,
    hInstance,
    NULL
);
  • WS_OVERLAPPEDWINDOW — standard window style
  • CW_USEDEFAULT — auto-position
ShowWindow & UpdateWindow 

ShowWindow(hwnd, nCmdShow);
UpdateWindow(hwnd);
These make the window visible and trigger initial painting.
The message loop Every WinAPI program has a message loop: 

MSG msg;

while (GetMessage(&msg, NULL, 0, 0)) {
    TranslateMessage(&msg);
    DispatchMessage(&msg);
}
  • GetMessage — waits for messages
  • TranslateMessage — keyboard translation
  • DispatchMessage — calls WndProc
Window procedure (WndProc) 

LRESULT CALLBACK WndProc(HWND hwnd, UINT msg,
                         WPARAM wParam, LPARAM lParam)
{
    switch (msg) {
        case WM_DESTROY:
            PostQuitMessage(0);
            return 0;

        case WM_PAINT:
            // drawing happens here
            break;
    }

    return DefWindowProc(hwnd, msg, wParam, lParam);
}
WndProc handles all window events.
Common WinAPI messages
  • WM_CREATE — window created
  • WM_PAINT — redraw window
  • WM_SIZE — window resized
  • WM_COMMAND — menu/button events
  • WM_DESTROY — window closed
WinAPI naming conventions WinAPI uses consistent naming:
  • HWND — window handle
  • HDC — device context
  • HINSTANCE — program instance
  • LPSTR — char*
  • LPVOID — void*
Understanding these is essential for all future chapters.
Minimal WinAPI program 

#include <windows.h>

LRESULT CALLBACK WndProc(HWND hwnd, UINT msg,
                         WPARAM wParam, LPARAM lParam)
{
    switch (msg) {
        case WM_DESTROY:
            PostQuitMessage(0);
            return 0;
    }
    return DefWindowProc(hwnd, msg, wParam, lParam);
}

int WINAPI WinMain(HINSTANCE hInstance,
                   HINSTANCE hPrevInstance,
                   LPSTR lpCmdLine,
                   int nCmdShow)
{
    WNDCLASSEX wc = {0};
    wc.cbSize        = sizeof(WNDCLASSEX);
    wc.lpfnWndProc   = WndProc;
    wc.hInstance     = hInstance;
    wc.lpszClassName = "MyWindowClass";

    RegisterClassEx(&wc);

    HWND hwnd = CreateWindowEx(
        0, "MyWindowClass", "WinAPI Window",
        WS_OVERLAPPEDWINDOW,
        CW_USEDEFAULT, CW_USEDEFAULT,
        800, 600,
        NULL, NULL, hInstance, NULL
    );

    ShowWindow(hwnd, nCmdShow);
    UpdateWindow(hwnd);

    MSG msg;
    while (GetMessage(&msg, NULL, 0, 0)) {
        TranslateMessage(&msg);
        DispatchMessage(&msg);
    }

    return 0;
}
