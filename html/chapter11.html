Chapter 11 — Advanced C Concepts & Preparation for WinAPI

Concept Description
Structs Structs group related variables into a single composite type. 

struct Player {
    char name[32];
    int score;
    float health;
};
Usage: 

struct Player p = {"Demon", 9001, 100.0f};
printf("%s: %d\\n", p.name, p.score);
Typedef Creates aliases for types: 

typedef unsigned long ulong;
typedef struct Player Player;
Makes code cleaner and easier to maintain.
Enums Enumerations define named integer constants: 

enum State { IDLE, RUNNING, JUMPING };
enum State s = RUNNING;
Useful for readable state machines.
Bitwise operations 

int flags = 0;
flags |= 1;   // set bit 0
flags |= 2;   // set bit 1
flags &= ~1;  // clear bit 0
Critical for low-level programming, drivers, and WinAPI flags.
Memory layout Understanding how C stores data prepares you for WinAPI structures. 

struct Vec2 {
    float x;
    float y;
};
Stored contiguously:
  • x at offset 0
  • y at offset 4
Dynamic memory (malloc/free) 

int *arr = malloc(sizeof(int) * 10);
if (!arr) return 1;

arr[0] = 123;

free(arr);
WinAPI uses similar patterns with HeapAlloc and LocalAlloc.
Realloc Resize dynamic memory: 

int *arr = malloc(5 * sizeof(int));
arr = realloc(arr, 10 * sizeof(int));
Useful for buffers and dynamic arrays.
Function pointers 

int add(int a, int b) { return a + b; }

int (*fp)(int, int) = add;
printf("%d\\n", fp(3, 4));
WinAPI uses callbacks extensively (WndProc, timers, hooks).
Callback functions 

void process(int (*fn)(int)) {
    printf("%d\\n", fn(5));
}
WinAPI message handlers follow similar patterns.
Preprocessor macros 

#define MAX(a,b) ((a) > (b) ? (a) : (b))
#define DEBUG 1

#if DEBUG
printf("Debug mode\\n");
#endif
WinAPI uses macros heavily for flags, constants, and message IDs.
Multiple source files 

// math.h
int add(int a, int b);

// math.c
int add(int a, int b) { return a + b; }

// main.c
#include "math.h"
printf("%d\\n", add(2, 3));
WinAPI projects always use multiple files.
Makefiles 

main: main.c math.c
    gcc main.c math.c -o main
Windows equivalents: nmake, MSBuild.
Understanding headers WinAPI relies on massive header files:
  • windows.h
  • winuser.h
  • wingdi.h
  • commctrl.h
Knowing how C headers work is essential.
Preparing for WinAPI WinAPI requires mastery of:
  • pointers
  • structs
  • function pointers
  • memory management
  • bitwise flags
You now have all the C fundamentals needed to build Windows applications.
Example: Combined advanced C program 

#include <stdio.h>
#include <stdlib.h>

typedef struct {
    char name[32];
    int score;
} Player;

void show(Player *p) {
    printf("%s: %d\\n", p->name, p->score);
}

int main(void) {
    Player *p = malloc(sizeof(Player));
    if (!p) return 1;

    p->score = 9001;
    snprintf(p->name, 32, "Demon");

    show(p);

    free(p);
    return 0;
}
