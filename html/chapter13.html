Chapter 13 — WinAPI Project Structure

Component Description
WinMain The entry point for Windows GUI applications. 

int WINAPI WinMain(HINSTANCE hInstance,
                   HINSTANCE hPrevInstance,
                   LPSTR lpCmdLine,
                   int nCmdShow)
{
    // initialization
    return 0;
}
WinMain replaces main() in GUI programs.
Window Class (WNDCLASSEX) Defines the behavior and appearance of windows created from the class. 

WNDCLASSEX wc = {0};
wc.cbSize        = sizeof(WNDCLASSEX);
wc.lpfnWndProc   = WndProc;
wc.hInstance     = hInstance;
wc.lpszClassName = "MyWindowClass";

RegisterClassEx(&wc);
Key fields:
  • lpfnWndProc — message handler
  • hInstance — program instance
  • lpszClassName — class name
CreateWindowEx Creates a window instance from the registered class. 

HWND hwnd = CreateWindowEx(
    0,
    "MyWindowClass",
    "My WinAPI Window",
    WS_OVERLAPPEDWINDOW,
    CW_USEDEFAULT, CW_USEDEFAULT,
    800, 600,
    NULL, NULL,
    hInstance,
    NULL
);
WS_OVERLAPPEDWINDOW gives a standard window frame.
ShowWindow / UpdateWindow Makes the window visible and triggers initial painting. 

ShowWindow(hwnd, nCmdShow);
UpdateWindow(hwnd);
Message Loop The heart of every WinAPI program. 

MSG msg;

while (GetMessage(&msg, NULL, 0, 0)) {
    TranslateMessage(&msg);
    DispatchMessage(&msg);
}
  • GetMessage — waits for messages
  • TranslateMessage — keyboard translation
  • DispatchMessage — calls WndProc
Window Procedure (WndProc) Handles all messages sent to the window. 

LRESULT CALLBACK WndProc(HWND hwnd, UINT msg,
                         WPARAM wParam, LPARAM lParam)
{
    switch (msg) {
        case WM_DESTROY:
            PostQuitMessage(0);
            return 0;

        case WM_PAINT:
            // drawing code
            break;
    }

    return DefWindowProc(hwnd, msg, wParam, lParam);
}
WndProc is the core of WinAPI logic.
Project File Layout A typical WinAPI project uses multiple files: 

/project
    main.c
    window.c
    window.h
    resources.rc
    Makefile (or MSBuild project)
  • main.c — WinMain + initialization
  • window.c — WndProc + window logic
  • window.h — declarations
  • resources.rc — icons, menus, dialogs
Resources (.rc file) WinAPI uses resource files for icons, menus, dialogs: 

IDI_ICON1 ICON "app.ico"
Compiled with rc.exe or windres.
Handles WinAPI uses handles to reference objects:
  • HWND — window
  • HDC — device context
  • HMENU — menu
  • HBRUSH — brush
  • HFONT — font
Handles are opaque — you never access their internals.
WinAPI Naming Conventions WinAPI uses consistent prefixes:
  • LPSTR — char*
  • LPVOID — void*
  • DWORD — 32-bit unsigned
  • UINT — unsigned int
  • BOOL — integer boolean
Understanding these is essential for reading documentation.
Minimal WinAPI Project Structure 

// main.c
#include <windows.h>
#include "window.h"

int WINAPI WinMain(HINSTANCE hInstance,
                   HINSTANCE hPrevInstance,
                   LPSTR lpCmdLine,
                   int nCmdShow)
{
    return CreateMainWindow(hInstance, nCmdShow);
}
      

// window.c
#include <windows.h>
#include "window.h"

LRESULT CALLBACK WndProc(HWND hwnd, UINT msg,
                         WPARAM wParam, LPARAM lParam)
{
    switch (msg) {
        case WM_DESTROY:
            PostQuitMessage(0);
            return 0;
    }
    return DefWindowProc(hwnd, msg, wParam, lParam);
}

int CreateMainWindow(HINSTANCE hInstance, int nCmdShow)
{
    WNDCLASSEX wc = {0};
    wc.cbSize        = sizeof(WNDCLASSEX);
    wc.lpfnWndProc   = WndProc;
    wc.hInstance     = hInstance;
    wc.lpszClassName = "MainWindow";

    RegisterClassEx(&wc);

    HWND hwnd = CreateWindowEx(
        0, "MainWindow", "WinAPI App",
        WS_OVERLAPPEDWINDOW,
        CW_USEDEFAULT, CW_USEDEFAULT,
        800, 600,
        NULL, NULL, hInstance, NULL
    );

    ShowWindow(hwnd, nCmdShow);
    UpdateWindow(hwnd);

    MSG msg;
    while (GetMessage(&msg, NULL, 0, 0)) {
        TranslateMessage(&msg);
        DispatchMessage(&msg);
    }

    return 0;
}
