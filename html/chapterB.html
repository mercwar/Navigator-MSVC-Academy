Chapter B — CYBORG: Command Bus & DirectX Routing

TopicDescription
CYBORG Role
  • CYBORG = command bus + AI/LLM‑driven execution layer.
  • Does not own DX; it routes to CBORD/MERC‑G.
  • Works off AVIS‑registered vector tables.
Command Object 

enum CmdType {
    CMD_DRAW_LINE,
    CMD_DRAW_TEXT,
    CMD_LOAD_TEXTURE,
    CMD_PRESENT
};

struct Cmd {
    CmdType type;
    void   *payload;
};
DX Routing via AVIS 

void CYBORG_Execute(const Cmd &cmd)
{
    DXVector *dx = &avis.cbord->dx;

    switch (cmd.type) {
        case CMD_DRAW_LINE:
            MERCG_DrawLine(dx, (LinePayload*)cmd.payload);
            break;
        case CMD_DRAW_TEXT:
            MERCG_DrawText(dx, (TextPayload*)cmd.payload);
            break;
        case CMD_PRESENT:
            dx->swap->Present(1, 0);
            break;
    }
}
LLM‑Driven Commands
  • LLM parses user intent → emits `Cmd` objects.
  • CYBORG never touches DX directly; it only calls MERC‑G/CBORD.
  • AVIS guarantees DXVector validity before execution.
