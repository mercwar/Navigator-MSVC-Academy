Chapter 3 — Compiling & Running C Programs

Stage Description
1. Preprocessing The preprocessor handles:
  • #include — inserts header files
  • #define — macros
  • #if / #ifdef — conditional compilation
Output: a single expanded .i file.
2. Compilation Converts preprocessed C code into assembly instructions. Output: .s assembly file. 

gcc -S main.c
3. Assembly Translates assembly into machine code. Output: .o object file. 

gcc -c main.c
4. Linking Combines object files and libraries into an executable. Output: a.out, main, or .exe. 

gcc main.o -o main
Full build (Linux/macOS) 

gcc main.c -o main
./main
  • gcc compiles and links automatically.
  • ./main runs the program.
Full build (Windows MSVC) 

cl main.c
main.exe
  • cl compiles and links.
  • main.exe is the final executable.
Compiler warnings Warnings help catch mistakes: 

gcc -Wall -Wextra main.c -o main
  • -Wall — enable common warnings
  • -Wextra — enable additional warnings
Debug builds Add debugging symbols: 

gcc -g main.c -o main
Use with gdb or lldb.
Optimized builds 

gcc -O2 main.c -o main
  • -O1 — basic optimization
  • -O2 — recommended
  • -O3 — aggressive optimization
Common errors
  • Undefined reference — missing library or function
  • Missing semicolon — syntax error
  • Implicit declaration — forgot a header
  • Segmentation fault — invalid memory access
