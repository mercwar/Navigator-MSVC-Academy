Chapter 22 — Processes & Inter‑Process Communication (IPC)

Topic Description
Creating a Process WinAPI creates processes using CreateProcess: 

STARTUPINFO si = {0};
si.cb = sizeof(si);

PROCESS_INFORMATION pi = {0};

CreateProcess(
    "C:\\Windows\\System32\\notepad.exe",
    NULL,
    NULL, NULL,
    FALSE,
    0,
    NULL, NULL,
    &si,
    &pi
);
Close handles when done: 

CloseHandle(pi.hProcess);
CloseHandle(pi.hThread);
Process Information PROCESS_INFORMATION contains:
  • hProcess — process handle
  • hThread — main thread handle
  • dwProcessId — PID
  • dwThreadId — TID
Synchronization Objects WinAPI provides multiple IPC synchronization primitives:
  • Events
  • Mutexes
  • Semaphores
  • Critical sections
Events Events signal between threads or processes: 

HANDLE hEvent = CreateEvent(NULL, FALSE, FALSE, "MyEvent");

SetEvent(hEvent);     // signal
ResetEvent(hEvent);   // clear

WaitForSingleObject(hEvent, INFINITE);
Named events allow cross‑process communication.
Mutex Mutexes ensure exclusive access: 

HANDLE hMutex = CreateMutex(NULL, FALSE, "MyMutex");

WaitForSingleObject(hMutex, INFINITE);
// critical section
ReleaseMutex(hMutex);
Semaphore Semaphores allow limited access: 

HANDLE hSem = CreateSemaphore(NULL, 3, 3, "MySem");

WaitForSingleObject(hSem, INFINITE);
// do work
ReleaseSemaphore(hSem, 1, NULL);
Named Pipes — Server Named pipes allow full duplex IPC. Server: 

HANDLE pipe = CreateNamedPipe(
    "\\\\.\\pipe\\mypipe",
    PIPE_ACCESS_DUPLEX,
    PIPE_TYPE_BYTE | PIPE_READMODE_BYTE | PIPE_WAIT,
    1,
    1024, 1024,
    0,
    NULL
);

ConnectNamedPipe(pipe, NULL);

char buffer[256];
DWORD read;
ReadFile(pipe, buffer, 256, &read, NULL);

WriteFile(pipe, "OK", 2, NULL, NULL);

DisconnectNamedPipe(pipe);
CloseHandle(pipe);
Named Pipes — Client 

HANDLE pipe = CreateFile(
    "\\\\.\\pipe\\mypipe",
    GENERIC_READ | GENERIC_WRITE,
    0,
    NULL,
    OPEN_EXISTING,
    0,
    NULL
);

WriteFile(pipe, "Hello", 5, NULL, NULL);

char buffer[256];
DWORD read;
ReadFile(pipe, buffer, 256, &read, NULL);

CloseHandle(pipe);
Shared Memory (File Mapping) Shared memory allows multiple processes to access the same RAM. Create: 

HANDLE hMap = CreateFileMapping(
    INVALID_HANDLE_VALUE,
    NULL,
    PAGE_READWRITE,
    0,
    4096,
    "SharedMem"
);
Map: 

char *data = (char*)MapViewOfFile(
    hMap,
    FILE_MAP_ALL_ACCESS,
    0, 0,
    0
);

strcpy(data, "Hello from process A");
Another process: 

HANDLE hMap = OpenFileMapping(
    FILE_MAP_ALL_ACCESS,
    FALSE,
    "SharedMem"
);

char *data = (char*)MapViewOfFile(
    hMap,
    FILE_MAP_ALL_ACCESS,
    0, 0,
    0
);

printf("%s\n", data);
Job Objects Job objects manage groups of processes: 

HANDLE hJob = CreateJobObject(NULL, "MyJob");

AssignProcessToJobObject(hJob, pi.hProcess);
Jobs allow:
  • CPU limits
  • Memory limits
  • Kill‑on‑close
Full IPC Example (Shared Memory + Event) 

#include <windows.h>
#include <stdio.h>

int main(void)
{
    HANDLE hMap = CreateFileMapping(
        INVALID_HANDLE_VALUE,
        NULL,
        PAGE_READWRITE,
        0,
        4096,
        "SharedBlock"
    );

    char *data = (char*)MapViewOfFile(
        hMap,
        FILE_MAP_ALL_ACCESS,
        0, 0, 0
    );

    HANDLE hEvent = CreateEvent(NULL, FALSE, FALSE, "SharedEvent");

    strcpy(data, "Hello from process A");

    SetEvent(hEvent);

    UnmapViewOfFile(data);
    CloseHandle(hMap);
    CloseHandle(hEvent);

    return 0;
}
