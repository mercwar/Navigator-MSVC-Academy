Chapter R — Monitoring

AreaRoutine
Apache Logs 

Location:
C:\Apache24\logs\error.log
C:\Apache24\logs\access.log

Routine:
- Tail logs with: type error.log
- Check access patterns daily
PHP Logs 

Edit php.ini:
log_errors = On
error_log = "C:/php/php_errors.log"

Routine:
- Review php_errors.log after deployments
MSVC Build Logs 

Enable verbose output:
cl /Wall /Fe:app.exe source.cpp > build.log 2>&1

Routine:
- Scan build.log for warnings/errors
MSYS2 Build Logs 

Use Ninja with logging:
ninja -j8 > ninja.log 2>&1

Routine:
- Review ninja.log for failed targets
GitHub CLI Monitoring 

Run:
gh issue list
gh pr list
gh run list

Routine:
- Track repo activity
- Monitor CI/CD runs
System Resource Monitoring 

CMD commands:
tasklist
systeminfo
wmic cpu get loadpercentage
wmic os get freephysicalmemory

Routine:
- Check CPU/memory before builds
