Chapter D — MERC‑G: DirectX Render Core & Subsystem Hooks

TopicDescription
MERC‑G Role
  • MERC‑G = DirectX render core for MercWar.
  • Owns pipelines: D2D, DWrite, D3D11.
  • Exposes high‑level draw functions to CYBORG.
Initialization from CBORD 

void MERCG_Init(DXVector *dx)
{
    MERCG_InitD2D(dx);
    MERCG_InitDWrite(dx);
    MERCG_InitD3D(dx);
}
Example Draw Line 

void MERCG_DrawLine(DXVector *dx, const LinePayload *p)
{
    ID2D1HwndRenderTarget *rt = MERCG_GetRT(dx);
    ID2D1SolidColorBrush  *b  = MERCG_GetBrush(dx, p->color);

    rt->BeginDraw();
    rt->DrawLine(
        D2D1::Point2F(p->x1, p->y1),
        D2D1::Point2F(p->x2, p->y2),
        b,
        p->thickness
    );
    rt->EndDraw();
}
Example Draw Text 

void MERCG_DrawText(DXVector *dx, const TextPayload *p)
{
    ID2D1HwndRenderTarget *rt = MERCG_GetRT(dx);
    IDWriteTextFormat      *fmt= MERCG_GetFormat(p->font, p->size);
    ID2D1SolidColorBrush   *b  = MERCG_GetBrush(dx, p->color);

    rt->BeginDraw();
    rt->DrawText(
        p->text,
        (UINT)wcslen(p->text),
        fmt,
        D2D1::RectF(p->x, p->y, p->x2, p->y2),
        b
    );
    rt->EndDraw();
}
Subsystem Hook Map
  • CBORD → creates DXVector.
  • MERC‑G → consumes DXVector, owns pipelines.
  • CYBORG → calls MERC‑G functions.
  • CYHY → monitors DXVector health.
